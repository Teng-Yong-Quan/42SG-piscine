# 42SG-piscine

Here are the answers to the respective questions.

Please take them with a pinch of salt, as they may not be entirely correct. Do note that not all questions have been attempted.

For the WSL Ubuntu terminal, the following tools may be useful during the 42 Piscine:

* **Visual Studio Code**
  Provides syntax highlighting, useful extensions, and convenient Git integration.
  * Some useful extensions:
	  * `42 ft count line` by DoKca
	  * `42 Header` by kube

* **Valgrind**
  Detects memory leaks and other memory-related issues, such as unfreed dynamically allocated memory.
  * Installation command:
  `sudo apt-get install valgrind`

* **Norminette**
  Checks whether your code complies with the 42 coding standards and conventions.
  * Website:
  `https://github.com/42school/norminette`

* **Mini Moulinette**
  A community-developed replica of the actual Moulinette. While useful for preliminary testing, it is not as strict or as accurate as the official Moulinette used by 42.
  * Website:
  `https://github.com/k11q/mini-moulinette`
